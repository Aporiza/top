step_oracle 参考路径

这条路径只在 CONFIG_BPU 关闭时进入。当前 large/default 配置定义了 CONFIG_BPU，所以正常主线仍是 step_bpu()。

一句话主线

Oracle 分支不进入 front_top()，而是在 front_cycle() 的非 BPU 分支里通过 1-entry pending 调 front.step_oracle()，再由 get_oracle() 用参考 CPU 直接生成 front.out。

和 step_bpu 主路径的区别

step_bpu 主路径

进入 front_top()，按 front_seq_read -> front_comb_calc -> front_seq_write 读旧状态、跑组合、统一提交状态。

step_oracle 参考路径

不进入 front_top()，不跑 BPU/FIFO/PTAB/ICache 前端流水，直接用 RefCpu oracle.exec() 产生理想取指输出。
