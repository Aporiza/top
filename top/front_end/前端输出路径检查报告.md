# 前端 Top 未体现项检查报告

## 1. 检查口径

本文档只记录 `top/front_end/front_top.v` 相比最新模拟器源码中“没有体现、只部分体现、或需要老板确认是否展开”的路径。

已经核对正确、来源清楚、且当前 `.v` 中已经连接到位的输出不再列入本文档，避免重复说明。

## 3. 未体现项汇总

| 序号 | 未体现项 | 源码位置 | 当前 `.v` 表现 | 建议处理 |
|---|---|---|---|---|
| 1 | `front2back` 到最终输出的 bypass 路径 | `front_top.cpp` 中 `can_bypass_front2back_to_output` 和 `front_output_comb` | 顶层输出只来自 `front2back_fifo_out` | 若要求周期行为更贴近 C++，需要补 bypass 选择；若只画主线，需要在说明中标注为未展开旁路。 |
| 2 | ICache 到 predecode/checker 的 bypass 路径 | `front_top.cpp` 中 `can_bypass_icache_to_predecode` | `predecode_checker` 输入只来自 `instruction_fifo_out` | 该路径受宏控制，当前未画；如果老板要求完整数据路径，需要补从 `icache_out` 直达 predecode/checker 的旁路。 |
| 3 | BPU fetch 地址直接送 ICache 的 bypass 路径 | `front_top.cpp` 中 `can_bypass_fetch_to_icache` | `icache_read_valid/fetch_address` 只来自 `fetch_address_fifo_out` | 该路径受宏控制，当前未画；如果要求体现旁路，需要补 BPU 到 ICache 的直接选择器。 |
| 4 | ICache 第二路请求/返回 `slot1` 主线 | `front_IO.h` 中 `icache_read_valid_2/fetch_address_2/fetch_group_2`，`front_top.cpp` 中 slot1 逻辑 | `icache_read_valid_2` 和 `fetch_address_2` 在顶层固定为 0，`fetch_group_2` 没有继续向后连接 | 当前配置默认关闭双请求，保留为未展开项；若新版要求双取指，需要展开第二路 fetch FIFO、ICache 返回和 instruction FIFO 写入。 |
| 5 | `ENABLE_2AHEAD` 对 fetch address FIFO 的第二次写入 | `front_top.cpp` 中 `bpu_output.two_ahead_target` 写入 | `bpu_out_two_ahead_target` 被引出但没有进入 FIFO 写路径 | 当前配置有 `FRONTEND_DISABLE_2AHEAD`，因此主线不展开；若老板要求画可选路径，需要增加 second write 或单独说明关闭依据。 |
| 6 | `predecode_flush` 的延迟恢复状态 | `front_top.cpp` 中 `predecode_refetch/predecode_refetch_address` 快照 | 当前 `global_refetch = refetch || checker_out_predecode_flush_enable`，是组合立即生效 | 若只做组合连线骨架可以保留；若要求对应 C++ 时序，需要补内部状态或说明该延迟寄存器留在实现阶段。 |
| 7 | predecode flush 时对 ICache 的 `invalidate_req` | `front_top.cpp` 中 `invalidate_req_in.invalidate_req = true` | `icache_in_bus` 的 `invalidate_req` 固定为 `1'b0` | 需要确认该 ICache invalidate 是否属于必须表达的控制路径；若需要，应该由 checker flush 引入。 |
| 8 | `seq_next_pc` 跨 cache line 的对齐修正 | `front_top.cpp` 中 `seq_next_pc` 与 `ICACHE_LINE_SIZE` mask 逻辑 | 当前只写成 `icache_fetch_address + FETCH_WIDTH * 4` | 如果要求贴近 C++ 组合逻辑，需要补 line mask；如果只画模块连线，应标注该细节未展开。 |
| 9 | ICache PTW / memory runtime 端口 | `front_module.h` 中 `icache_set_ptw_mem_port`、`icache_set_ptw_walk_port`、`icache_set_mem_read_port` | 当前前端顶层没有对外 PTW、AXI/memory 端口 | 需要老板确认是否继续放在 ICache 内部不对外，还是前端 top 需要暴露外部存储接口。 |

## 4. 具体佐证

### 4.1 front2back 到最终输出的 bypass 未体现

源码中，当前拍如果 `front2back FIFO` 为空、后端正在读、并且本拍前端刚好可写，则可以不经过 FIFO 存储，直接用 `bypass_front2back_fifo_out` 作为最终输出来源。

```cpp
bool can_bypass_front2back_to_output =
    front2back_read_enable && rd.front2back_fifo_empty_latch_snapshot &&
    !saved_front2back_fifo_out.front2back_FIFO_valid && front2back_can_write;

front2back_fifo_in.write_enable =
    front2back_can_write && !can_bypass_front2back_to_output;

if (!saved_front2back_fifo_out.front2back_FIFO_valid &&
    use_front2back_output_bypass) {
    out_src = &bypass_front2back_fifo_out;
}
```

当前 `.v` 中最终输出固定来自 `front2back_fifo_out`，没有 `saved/bypass` 二选一。

```verilog
assign FIFO_valid = front2back_fifo_out_valid;
assign pc = front2back_fifo_out_predict_base_pc;
assign instructions = front2back_fifo_out_fetch_group;
assign predict_dir_out = front2back_fifo_out_predict_dir_corrected;
```

### 4.2 ICache 到 predecode/checker 的 bypass 未体现

源码中，若 instruction FIFO 为空、PTAB 有数据、front2back FIFO 未满，且 ICache 本拍有效，可以绕过 instruction FIFO，直接形成 `bypass_fifo_out` 进入后续 predecode/checker 逻辑。

```cpp
bool can_bypass_icache_to_predecode =
    rd.fifo_empty_latch_snapshot && !rd.ptab_empty_latch_snapshot &&
    !rd.front2back_fifo_full_latch_snapshot && !global_reset &&
    !global_refetch && icache_slot0_data_valid;

if (can_bypass_icache_to_predecode) {
    bypass_fifo_out.instructions[i] = icache_out.fetch_group[i];
    bypass_fifo_out.pc[i] = icache_out.fetch_pc + (i * 4);
    saved_fifo_out = bypass_fifo_out;
}
```

当前 `.v` 的 checker 输入只来自 `instruction_fifo_out`，没有从 `icache_out` 直接进入 checker 的旁路。

```verilog
assign predecode_checker_in_bus = {
    ptab_out_predict_dir,
    ptab_out_predict_next_fetch_address,
    instruction_fifo_out_predecode_type,
    instruction_fifo_out_predecode_target_address,
    instruction_fifo_out_seq_next_pc
};
```

### 4.3 BPU fetch 地址到 ICache 的直接 bypass 未体现

源码中，如果 fetch address FIFO 当前没有有效输出，但 BPU 本拍产生了新地址，并且 ICache ready，则可以直接把 BPU 地址送给 ICache。

```cpp
can_bypass_fetch_to_icache =
    !saved_fetch_addr_fifo_out_0.read_valid && normal_write_enable &&
    !bpu_output.mini_flush_correct && icache_ready && !global_refetch;

if (saved_fetch_addr_fifo_out_0.read_valid) {
    icache_in.fetch_address = saved_fetch_addr_fifo_out_0.fetch_address;
} else if (can_bypass_fetch_to_icache) {
    icache_in.fetch_address = bpu_output.fetch_address;
}
```

当前 `.v` 中 ICache 只接收 fetch address FIFO 输出。

```verilog
assign icache_read_valid =
    fetch_address_fifo_out_read_valid;
assign icache_fetch_address =
    fetch_address_fifo_out_fetch_address;
```

### 4.4 ICache slot1 双请求路径未展开

源码结构中 ICache 输入和输出包含第二路请求/返回。

```cpp
struct icache_in {
  wire1_t icache_read_valid_2;
  fetch_addr_t fetch_address_2;
};

struct icache_out {
  inst_word_t fetch_group_2[FETCH_WIDTH];
  wire1_t page_fault_inst_2[FETCH_WIDTH];
  wire1_t inst_valid_2[FETCH_WIDTH];
  pc_t fetch_pc_2;
};
```

`front_top.cpp` 中只有在 `FRONTEND_IDEAL_ICACHE_DUAL_REQ_ACTIVE` 打开时才使用第二路。

```cpp
#if FRONTEND_IDEAL_ICACHE_DUAL_REQ_ACTIVE
if (saved_fetch_addr_fifo_out_1.read_valid) {
    icache_in.icache_read_valid_2 = true;
    icache_in.fetch_address_2 = saved_fetch_addr_fifo_out_1.fetch_address;
}
#endif
```

当前 `.v` 中第二路请求固定为 0，第二路输出只在 `icache_top.v` 内拆出，没有继续接到 instruction FIFO。

```verilog
assign icache_in_bus = {
    global_reset,
    global_refetch,
    itlb_flush,
    fence_i,
    1'b0,
    icache_read_valid,
    icache_fetch_address,
    1'b0,
    32'b0,
    csr_status,
    1'b0
};
```

### 4.5 2-ahead fetch address FIFO 写入未展开

源码中 `ENABLE_2AHEAD` 打开时，BPU 会把 `two_ahead_target` 额外写入 fetch address FIFO。

```cpp
#ifdef ENABLE_2AHEAD
if (normal_write_enable) {
    fetch_addr_fifo_in.write_enable = true;
    fetch_addr_fifo_in.fetch_address = bpu_output.two_ahead_target;
}
#endif
```

当前 `.v` 中 `bpu_out_two_ahead_target` 只作为 BPU wrapper 输出引出，没有接入 fetch address FIFO 写入路径。

```verilog
assign fetch_address_fifo_write_address =
    fetch_address_fifo_write_enable ? bpu_out_fetch_address : 32'b0;
```

### 4.6 predecode flush 延迟恢复状态未体现

源码中 predecode checker 产生 flush 后，不是直接在同一个组合表达式里作为 `global_refetch`，而是写入 `predecode_refetch` 快照，下一轮由全局控制读取。

```cpp
static bool predecode_refetch = false;
static uint32_t predecode_refetch_address = 0;

output.global_refetch =
    input.backend_refetch || input.predecode_refetch_snapshot;

predecode_refetch = req.front_state.next_predecode_refetch;
predecode_refetch_address = req.front_state.next_predecode_refetch_address;
```

当前 `.v` 中为组合立即合并。

```verilog
assign global_refetch = refetch || checker_out_predecode_flush_enable;
assign global_refetch_address =
    refetch ? refetch_address :
    checker_out_predict_next_fetch_address_corrected;
```

### 4.7 ICache invalidate 控制未体现

源码中 predecode flush 发生时，会额外向 ICache 发一个 `invalidate_req`。

```cpp
if (do_predecode_flush) {
    struct icache_in invalidate_req_in;
    memset(&invalidate_req_in, 0, sizeof(invalidate_req_in));
    invalidate_req_in.invalidate_req = true;
    invalidate_req_in.csr_status = in->csr_status;
    icache_comb_calc(&invalidate_req_in, &icache_out);
}
```

当前 `.v` 中 `invalidate_req` 在 ICache 输入总线内固定为 0。

```verilog
assign icache_in_bus = {
    global_reset,
    global_refetch,
    itlb_flush,
    fence_i,
    1'b0,
    icache_read_valid,
    icache_fetch_address,
    1'b0,
    32'b0,
    csr_status,
    1'b0
};
```

### 4.8 `seq_next_pc` 的 cache line 对齐修正未体现

源码中 `seq_next_pc` 不是简单加 `FETCH_WIDTH * 4`，还会根据 `ICACHE_LINE_SIZE` 做跨 line 对齐修正。

```cpp
uint32_t mask = ~(ICACHE_LINE_SIZE - 1);
fifo_in.seq_next_pc = icache_out.fetch_pc + (FETCH_WIDTH * 4);
if ((fifo_in.seq_next_pc & mask) != (icache_out.fetch_pc & mask)) {
    fifo_in.seq_next_pc &= mask;
}
```

当前 `.v` 中只保留了主线加法。

```verilog
assign instruction_fifo_seq_next_pc =
    icache_fetch_address + (FETCH_WIDTH * 32'd4);
```

### 4.9 ICache PTW / memory runtime 端口未进入前端顶层

源码中 ICache 有运行时绑定的 PTW 和 memory read port。

```cpp
void icache_set_ptw_mem_port(PtwMemPort *port);
void icache_set_ptw_walk_port(PtwWalkPort *port);
void icache_set_mem_read_port(axi_interconnect::ReadMasterPort_t *port);
```

当前 `front_top.v` 没有暴露这些外部端口。是否继续放在 ICache 内部，还是需要前端 top 对外画出，需要老板确认。

## 5. 当前结论

当前 `front_top.v` 的主线连接已经可以表达 BPU、fetch address FIFO、ICache、instruction FIFO、PTAB、predecode checker、front2back FIFO 到后端输出的主要路径。

未体现的主要是旁路路径、可选双请求/2-ahead 路径、predecode flush 的时序状态、ICache invalidate 细节、`seq_next_pc` line mask 细节，以及 ICache 对外 PTW/memory runtime 端口。

如果老板要求的是“只画当前主线 shared front-end 数据流”，上述项目可以保留为未展开说明。如果要求更接近 C++ 周期行为，则优先建议补 `front2back output bypass`、`icache to predecode bypass`、`fetch to icache bypass`、`invalidate_req` 和 `seq_next_pc mask`。
