simulator-front step_oracle 理想预测路径说明

当前 RTL 交付口径是 BPU 一定在；Oracle 只作为模拟器里的理想 BPU/参考预测路径说明，不接入 front_top 硬件主线。

版本口径

simulator-front 版本的 front_top_out 没有 commit_stall，后端 FrontPreIO 也没有 front_stall。旧版文档里把 Oracle 写成固定 commit_stall=false 或 front_stall=false 的说法不适用于本次接口计数。

入口

主文件
simulator-front/rv_simu_mmu_v2.cpp:639-779
Oracle 调用
front.step_oracle() 位于 rv_simu_mmu_v2.cpp:732，对应模拟器里的理想预测参考路径。
函数定义
front-end/FrontTop.cpp:41-44 调用 get_oracle(in, out)。

执行顺序

  1. front_cycle() 先写 front.in.refetch/itlb_flush/fence_i/refetch_address。
  2. 如果没有 pending，则调用 front.step_oracle()，并把输出保存到 oracle_pending_out。
  3. 如果有 pending，则复用上一拍保存的 oracle_pending_out，避免后端 stall 丢包。
  4. 再把 front.out 的 valid、PC、inst、predict、SC、loop、TAGE 元信息逐 lane 写入 back.in。
  5. 如果未定义 CONFIG_ORACLE_STEADY_FETCH_WIDTH，taken 后面的 lane 会被 no_taken 屏蔽；本次 simulator-front 配置定义了该宏，Oracle 理想预测路径会保持固定取宽。

与 BPU 主线的关系

本次交付的前端训练版以 CONFIG_BPU 主线为准：front_top.v 实例化 bpu_top，bpu_top 再展开 13 个 BPU comb wrapper。Oracle 只保留为理想 BPU/参考预测说明，不参与当前端口宽度、模块拆分或可综合 RTL 连接。
